# 周计划看板 交互优化设计

> 日期：2026-06-09 | 状态：已确认

## 概述

在现有功能基础上，对前端进行组件化重构，并系统性优化交互体验。聚焦三个维度：

1. **操作效率** — 拖拽排序、键盘快捷键
2. **状态感知** — 日期芯片状态指示、保存动效、任务进度条
3. **流程连贯** — 昨日继承增强、收尾动画、周任务双向联动

## 组件架构

当前单文件 `index.html`（~277 行）拆分为 8 个组件，`index.html` 只保留 shell + 组合逻辑：

```
index.html (shell, ~80行)
├── WeekNav        — 周导航栏（◀ 周芯片 ▶）
├── WeekPlan       — 周计划编辑区（主线/必须/推进/临时）
├── DayTabs        — 日期芯片条（含状态指示）
├── YesterdayRef   — 昨日参考 + 智能继承提示
├── TaskPool       — 周任务池（拖拽排序、进度条、勾选）
├── DailyFocus     — 今日重点（拖拽排序、完成/迁移操作）
├── DailyReview    — 收尾区（四栏 + 明日优先）
└── WeekReview     — 周复盘（日记录/周总结双标签）
```

**组件通信：** 父子 props down，事件 `$emit` up。共享数据（当前周、当前日、日/周数据）在 shell 层管理，通过 props 下发。拖拽、快捷键等通用交互抽为 composable 函数。

**依赖：** SortableJS（CDN 引入，~12KB gzipped），无其他新依赖。

## 交互改进明细

### 1. 周计划逐条编辑

**现状：** 4 个 textarea，每行一条，无法单独操作、排序靠剪切粘贴。

**改进：** 统一为与"今日重点"一致的逐条编辑交互：

- 每条一个独立输入框
- 左侧⠿手柄拖拽排序
- 右侧✕按钮删除
- 底部"+ 添加"按钮新增
- 失焦自动保存（已有机制复用）
- 4 个区域（主线/必须/推进/临时）全部统一

**排序持久化：** 拖拽结束后按当前顺序将条目用 `\n` 拼接写入飞书多维表对应字段。无需新增 API。

### 2. 日期芯片状态指示

每个日期芯片显示 3 种状态：

| 状态 | 视觉 | 说明 |
|------|------|------|
| 已收尾 | 🟢绿色圆点 + "已收尾 ✓" | 今日已关闭 |
| 有内容（未收尾） | 🟡黄色圆点 + "有内容" | 填写了但没收尾 |
| 未填写/未来 | 无圆点，芯片半透明 | 空或还没到 |

**实现：** `DayTabs` 组件加载时请求 `/api/weeks/{wk}/days`，按日期匹配收尾状态和内容有无，计算每个芯片的 status class。今天额外显示"今天"标签 + 边框高亮。

### 3. 键盘快捷键

全局 keydown 监听，输入框聚焦时不拦截单键（仅拦截组合键）：

| 快捷键 | 操作 | 适用场景 |
|--------|------|----------|
| `←` / `→` | 切换前一天/后一天 | 非输入框聚焦时 |
| `Ctrl+S` | 保存日计划 + 周计划 | 全局 |
| `Ctrl+Enter` | 一键收尾当前日 | 全局 |
| `Ctrl+N` | 根据当前焦点区域新增条目 | 今日重点/任务池/周计划 |
| `Ctrl+I` | 继承昨日内容 | 今日重点区 |

**边界处理：** 输入框内不拦截单键。`Ctrl+S` 阻止浏览器默认保存行为。`Ctrl+Enter` 在收尾后显示 toast 确认。

### 4. 拖拽排序

**任务池：** SortableJS 绑定任务列表，拖拽结束后调用 `PUT /api/weeks/{wk}/tasks/reorder`（已有 API），按新顺序更新每条记录的排序字段。

**今日重点：** 拖拽结束后将 `focus` 数组按新顺序拼接为 `\n` 分隔文本，通过现有 `PUT /api/weeks/{wk}/days/{date}` 保存。

**周计划条目：** 同上，拖拽后拼接保存。

**视觉反馈：** 拖拽中的条目显示虚线边框 + 轻微阴影，松手后平滑归位。

### 5. 流程连贯

**昨日继承增强：**
- 进入新的一天时，自动请求昨日数据
- 如昨日有"明日优先"或"收尾_迁移"未处理，`YesterdayRef` 组件显示黄色提示条"昨日有 N 条待继承"
- 点击一键继承，内容并入今日重点，toast 反馈

**一键收尾动画：**
- 点击"一键收尾"后，今日重点每条依次飞入完成区（~200ms 间隔）
- 全部完成后日期芯片变绿，显示 "🎉 收尾完成！"
- 未完成的重点自动迁移到完成区

**周任务双向联动：**
- 勾选完成周任务 → 自动追加一条到今日收尾_完成区
- "临时进入" → 如含新条目，自动添加到任务池
- 当日完成区条目可"升级为周任务"（在完成区条目旁加按钮）

### 6. 保存反馈动效

- 点击保存/失焦自动保存后，保存按钮短暂变为绿色 "✓ 已保存"
- 2 秒后恢复原状
- 周计划和日计划各自独立反馈

### 7. 任务进度条

`TaskPool` 顶部显示本周任务完成进度：

```
📊 任务进度 3/5  ████████░░░░░░░░  60%
```

按类型分色显示（必须完成=橙、推进即可=黄、临时进入=红），数据从已有 API 返回的完成状态计算。

## 数据流

```
Shell (index.html)
  ├─ data: { wk, day, weekData, dayData, tasks, dayStatuses }
  ├─ methods: { saveWeek, saveDay, loadAll, switchDay, ... }
  │
  ├─→ WeekNav     props: { wk, weekOpts }        events: @switch-week, @create-week
  ├─→ WeekPlan    props: { weekData }             events: @save
  ├─→ DayTabs     props: { wk, day, dayStatuses } events: @switch-day
  ├─→ YesterdayRef props: { wk, day }             events: @inherit
  ├─→ TaskPool    props: { wk, tasks }            events: @toggle, @delete, @add, @reorder
  ├─→ DailyFocus  props: { dayData }              events: @save, @move, @reorder
  ├─→ DailyReview props: { dayData }              events: @save, @close
  └─→ WeekReview  props: { wk, weekData }         events: @save, @export
```

**API 调用：** 绝大多数交互改进复用现有后端 API。仅需新增 1 个路由：

| 方法 | 路径 | 说明 |
|------|------|------|
| `PUT` | `/api/weeks/{wk}/tasks/reorder` | 批量更新任务排序（body: `["id1","id2",...]`），后端 `reorder_tasks` 函数已存在，仅需暴露路由 |

## 错误处理

- **网络异常：** toast 提示"网络错误，请稍后重试"，操作回滚（如勾选失败恢复原状态）
- **拖拽冲突：** 同一时间只允许一个拖拽上下文，SortableJS 内置 handle 限制
- **键盘冲突：** 输入框内不拦截单键，仅组合键生效
- **空状态：** 无任务时进度条显示"暂无任务"，任务池显示引导文案
